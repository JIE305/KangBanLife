# 部署指南

## 推荐方案：Nginx 反向代理 + Gunicorn/Waitress + Vite 构建产物

```
浏览器
  │
  ▼
Nginx (:80/:443)
  ├── /api/*      → Gunicorn/Waitress (:5000)  # 后端 API
  ├── /uploads/*  → 静态文件目录                  # 用户上传
  └── /*          → dist/ 静态文件                # 前端 SPA
```

---

## 一、后端部署

### 1.1 安装生产级 WSGI 服务器

Flask 自带的 `debug=True` 服务器不适合生产环境。推荐：

```bash
# 选项 A：Waitress（Windows 友好，纯 Python）
pip install waitress

# 选项 B：Gunicorn（Linux/Mac）
pip install gunicorn
```

### 1.2 使用 Waitress 启动（Windows）

创建 `wsgi.py`：

```python
from app import create_app
from waitress import serve

app = create_app()
serve(app, host='0.0.0.0', port=5000, threads=4)
```

```bash
python wsgi.py
```

### 1.3 使用 Gunicorn 启动（Linux）

```bash
gunicorn -w 4 -b 0.0.0.0:5000 "app:create_app()"
```

参数说明：
- `-w 4`: 4 个 worker 进程
- `-b`: 监听地址和端口
- 可通过 `--access-logfile` 指定访问日志

### 1.4 环境变量管理

生产环境创建 `.env`：

```env
ARK_API_KEY=生产环境API密钥
ARK_ENDPOINT_ID=生产环境端点ID
JWT_SECRET_KEY=强随机字符串（至少32位）
```

> JWT_SECRET_KEY 务必更换为强随机字符串，否则 token 可被伪造。

## 二、前端部署

### 2.1 构建

```bash
cd KangBan-vue3
npm run build
```

产物在 `dist/` 目录：`index.html` + `assets/` （JS + CSS）。

### 2.2 构建前注意事项

- `vite.config.js` 中的 `host` 和 `proxy` 仅在开发模式生效，构建后前端直接调用 `/api/v1` 相对路径，由 Nginx 代理。
- 如果 API 部署在不同域名，构建前需设置 `VITE_API_BASE_URL` 环境变量。

## 三、Nginx 配置

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /path/to/KangBan-vue3/dist;
    index index.html;

    # API 代理到后端
    location /api/ {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 60s;
    }

    # 上传文件
    location /uploads/ {
        alias /path/to/KangBan-backend/uploads/;
    }

    # SPA 回退（Vue Router history 模式）
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 常用命令

```bash
# 测试配置
nginx -t

# 重载
nginx -s reload

# 启动
nginx
```

## 四、Docker 化（可选）

### 后端 Dockerfile

```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt waitress
COPY . .
EXPOSE 5000
CMD ["python", "wsgi.py"]
```

### 前端 Dockerfile（多阶段构建）

```dockerfile
# 构建阶段
FROM node:18-alpine AS build
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build

# 运行阶段
FROM nginx:alpine
COPY --from=build /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/conf.d/default.conf
EXPOSE 80
```

### docker-compose.yml

```yaml
version: '3.8'
services:
  backend:
    build: ./KangBan-backend
    ports:
      - "5000:5000"
    env_file: ./KangBan-backend/.env
    volumes:
      - ./KangBan-backend/uploads:/app/uploads

  frontend:
    build: ./KangBan-vue3
    ports:
      - "80:80"
    depends_on:
      - backend
```

## 六、安全检查清单

- [ ] JWT_SECRET_KEY 已更换为强随机字符串
- [ ] ARK_API_KEY 不提交至 Git（.env 已在 .gitignore 中）
- [ ] CORS 限制为生产域名（非 `*`）
- [ ] MAX_CONTENT_LENGTH 已配置（默认 5MB）
- [ ] Nginx 限制请求体大小（`client_max_body_size 5m;`）
- [ ] 防火墙仅开放 80/443 端口（5000 端口不对公网开放）
- [ ] Flask 不再以 `debug=True` 运行
- [ ] HTTPS 已启用
