# 数据库迁移计划：SQLite → MySQL

## 背景

项目原计划使用 MySQL 数据库，当前开发阶段使用 SQLite 作为轻量替代。迁移目标是将 `schema_lite.sql`（6 表）和后续的 `schema.sql`（15 表）完整迁移到 MySQL。

## 当前状态

| 现状 | 目标 |
|------|------|
| `sqlite3` 直接连接（无 ORM） | MySQL + PyMySQL / mysql-connector-python |
| `utils.py` 中 `get_db()` 每次新建连接 | 连接池（DBUtils 或 SQLAlchemy 连接池） |
| 建表 SQL 使用 SQLite 语法 | MySQL 兼容语法 |
| 无迁移工具 | 考虑 Alembic 或手动 SQL 迁移 |
| 数据库文件：`database.db`（98KB） | MySQL 数据库实例 |

## 迁移步骤

### 第 1 步：安装 MySQL 驱动

```bash
pip install pymysql   # 推荐，纯 Python 实现
# 或 pip install mysql-connector-python
pip install dbutils   # 连接池
```

更新 `requirements.txt` 添加 `pymysql` 和 `dbutils`。

### 第 2 步：改造 `config.py`

```python
import os

class Config:
    BASE_DIR = os.path.abspath(os.path.dirname(__file__))

    # 原 SQLite 配置（删除或注释）
    # DATABASE = os.path.join(BASE_DIR, 'database.db')

    # 新 MySQL 配置
    MYSQL_HOST = os.getenv('MYSQL_HOST', '127.0.0.1')
    MYSQL_PORT = int(os.getenv('MYSQL_PORT', 3306))
    MYSQL_USER = os.getenv('MYSQL_USER', 'unihealth')
    MYSQL_PASSWORD = os.getenv('MYSQL_PASSWORD', '')
    MYSQL_DATABASE = os.getenv('MYSQL_DATABASE', 'unihealth')
    MYSQL_CHARSET = 'utf8mb4'

    # 连接池配置
    DB_POOL_MIN = 2
    DB_POOL_MAX = 10

    # JWT / 上传配置不变...
```

### 第 3 步：改造 `utils.py` — `get_db()`

```python
import pymysql
from dbutils.pooled_db import PooledDB
from config import Config

_pool = None

def _get_pool():
    global _pool
    if _pool is None:
        _pool = PooledDB(
            creator=pymysql,
            mincached=Config.DB_POOL_MIN,
            maxconnections=Config.DB_POOL_MAX,
            host=Config.MYSQL_HOST,
            port=Config.MYSQL_PORT,
            user=Config.MYSQL_USER,
            password=Config.MYSQL_PASSWORD,
            database=Config.MYSQL_DATABASE,
            charset=Config.MYSQL_CHARSET,
            cursorclass=pymysql.cursors.DictCursor
        )
    return _pool

def get_db():
    return _get_pool().connection()
```

> `DictCursor` 返回 `dict` 而非 `tuple`，与当前 `sqlite3.Row` 行为一致，`row_to_dict()` 可保持不变。

### 第 4 步：SQL 语法差异对照

| SQLite | MySQL | 说明 |
|--------|-------|------|
| `INTEGER PRIMARY KEY AUTOINCREMENT` | `INT AUTO_INCREMENT PRIMARY KEY` | 自增主键 |
| `DATETIME DEFAULT CURRENT_TIMESTAMP` | `DATETIME DEFAULT CURRENT_TIMESTAMP` | 相同，MySQL 需 5.6.5+ |
| `TINYINT DEFAULT 0` | `TINYINT DEFAULT 0` | 相同 |
| `VARCHAR(20)` | `VARCHAR(20)` | 相同 |
| `DECIMAL(5,2)` | `DECIMAL(5,2)` | 相同 |
| `TEXT` | `TEXT` | 相同 |
| `PRAGMA foreign_keys = ON` | 无需（MySQL 默认启用） | MySQL 需要 InnoDB 引擎 |
| `sqlite_sequence` 重置 | `ALTER TABLE ... AUTO_INCREMENT = 1` | 种子数据脚本需要 |
| `MAX(0, likes - 1)` | `GREATEST(0, likes - 1)` | MySQL 用 GREATEST |
| `CREATE UNIQUE INDEX ...` | 语法一致 | 相同 |

## 风险点

| 风险 | 缓解措施 |
|------|---------|
| `MAX()` vs `GREATEST()` 等函数差异遗漏 | 迁移前全文搜索 SQL 内置函数 |
| SQLite `sqlite3.Row` 与 `DictCursor` 行为差异 | 验证 `row_to_dict()` 返回格式一致 |
| 连接池泄漏 | 确保每个 `get_db()` 调用后正确关闭连接 |
| MySQL 服务不可用时应用崩溃 | 添加连接失败重试和降级日志 |
| 生产数据丢失 | 迁移前备份 `database.db` |

## 建议

- **开发阶段可暂缓迁移**，SQLite 对单机开发足够且零配置。
- **集成测试阶段应切换到 MySQL**，因为 SQLite 的某些行为（如并发锁、类型强制）与 MySQL 不同。
- 如果未来使用 ORM（如 SQLAlchemy），迁移成本会大幅降低。当前原始 SQL 架构下建议先迁移再考虑 ORM。
