## 已知问题清单

### 前端（14 项）
1. **package.json 缺少 test 脚本** — 测试文件存在但 `npm test` 不可用
2. **app.js Store 未被使用** — 定义了导航和功能卡片数据但任何组件都不引用
3. **health.js Store 未被使用** — Health.vue 自行实现了所有饮水/BMI/清单逻辑，完全不引用 Pinia health store（与 app.js 一样是冗余代码）
4. **Resources.vue 搜索框无效** — UI 存在但按键未绑定任何筛选逻辑
5. **Sports.vue 课程标签无效** — "全部/拉伸/力量" 标签为纯装饰
6. **ArticleDetail.vue 正文占位** — 始终显示"在此输入文章内容"
7. **首页"查看全部资讯"无路由** — 链接是普通 `<a>` 标签无 `href`
8. **高德 API Key 硬编码** — index.html 中以明文暴露
9. **Token 双存储** — 同时存 sessionStorage 和 localStorage，多标签页可能不一致
10. **Navbar 与 userStore 认证逻辑重叠** — 两处独立读取 storage，可能分歧
11. **ChatPanel 访客聊天无大小限制** — localStorage 存储无限增长
12. **冥想音频外链** — 指向外部 envato 托管 URL，可能失效
13. **登录页背景动画性能** — 12 个 requestAnimationFrame 圆形无防抖
14. **记忆登录与路由守卫不一致** — 守卫读 sessionStorage，Navbar 读 localStorage

### 后端（4 项）
1. **schema.sql 与 schema_lite.sql 并存** — 两套建表脚本，容易混淆
2. **resources 蓝图查询未建表** — resources.py 查询 first_aid/medicines/facilities 但 schema_lite 无对应表
3. **数据库连接无连接池** — 每次请求新建 sqlite3 连接
4. **错误处理不统一** — chat.py 的 try-except 吞异常，其他地方直接抛

## 与完整版设计的差距

plan/ 中的完整版文档定义了：
- **30 个 API 端点**（实际实现 ~14 个含 chat）
- **15 张数据库表**（实际建表 6 张）
- **9 个功能模块**（实际 4 个有后端支持：auth/user/articles/posts，chat + resources 部分可用）

差距主要在 5 个模块：health、meals、sports、resources（完整版）、mood。详见 [第二阶段实施计划](../第二阶段实施计划.md)。
