# 快速启动

## 环境要求

- Python 3.10+
- Node.js 18+
- npm 9+

## 1. 启动后端（端口 5000）

```bash
cd KangBan-backend

# 首次：创建虚拟环境并安装依赖
python -m venv venv
source venv/Scripts/activate  # Windows Git Bash
# 或 venv\Scripts\activate   # Windows CMD
pip install -r requirements.txt

# 配置环境变量（复制模板修改）
cp .env.example .env
# 编辑 .env，至少填入：
#   ARK_API_KEY=你的火山引擎API密钥（AI聊天功能需要）
#   ARK_ENDPOINT_ID=你的端点ID
#   JWT_SECRET_KEY=随机字符串

# 初始化数据库并填充种子数据
python seed_data.py

# 启动
python app.py
# → 服务运行在 http://127.0.0.1:5000
```

## 2. 启动前端（端口 3000）

```bash
cd KangBan-vue3

# 安装依赖
npm install

# 开发模式启动
npm run dev
# → 访问 http://localhost:3000
```

前端开发服务器自动将 `/api` 和 `/uploads` 请求代理到 `http://127.0.0.1:5000`。

## 3. .env 配置要点

后端 `KangBan-backend/.env`：

```env
# 火山引擎 ARK（豆包 AI）— 不填则聊天功能回退到规则引擎
ARK_API_KEY=ark-xxx
ARK_ENDPOINT_ID=ep-xxx

# JWT 密钥 — 生产环境务必更换
JWT_SECRET_KEY=随机字符串
```

前端无需 .env，API 地址通过 Vite 代理配置在 `vite.config.js` 中。

## 5. 常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| 前端页面无文章数据 | 后端未启动或未执行 seed_data | 启动后端 + `python seed_data.py` |
| 登录失败 401 | 用户不存在或密码错误 | 使用种子账号 2024001/123456 |
| AI 聊天总回复规则引擎内容 | ARK_API_KEY 未配置或过期 | 检查 .env 中的 API 密钥 |
| AI 聊天不保存历史 | 未登录（游客模式不持久化） | 登录后再聊天 |
| 头像上传失败 | uploads/ 目录不存在 | 后端会自动创建 |
| 前端 404 | 路由路径错误 | 检查 URL 是否与路由表匹配 |
| CORS 错误 | 前端端口非 3000/5173 | 检查 app.py 中 CORS 允许的源 |
| `pip install` 失败 | 虚拟环境未激活 | 先 `source venv/Scripts/activate` |
