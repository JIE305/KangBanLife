# 文件结构

## 后端 `KangBan-backend/`

```
KangBan-backend/
├── app.py              ★ 应用工厂，注册蓝图，初始化数据库
├── auth.py             ★ 认证蓝图：登录/注册/登出，JWT 签发
├── config.py           ★ 配置中心：读取 .env，JWT/数据库/上传目录配置
├── utils.py            ★ 工具函数：get_db(), success(), error(), 分页包装, 时间格式化
├── seed_data.py          种子数据脚本：独立运行，填充测试用户/文章/帖子
├── requirements.txt      依赖清单：flask, flask-cors, flask-jwt-extended, requests 等
├── .gitignore            忽略 .env, __pycache__, venv, *.db, uploads
├── .env                  环境变量（含 API 密钥，不提交 Git）
├── .env.example          环境变量模板
├── schema.sql            完整版建表 SQL（15 表，含未实现模块）
├── schema_lite.sql       精简版建表 SQL（6 表，当前实际使用）
├── database.db           SQLite 数据库文件
│
├── api/
│   ├── __init__.py       空，包标记
│   ├── articles.py       ★ 文章蓝图：列表/详情/精选，公开访问
│   ├── posts.py          ★ 帖子蓝图：CRUD + 点赞，部分需认证
│   ├── chat.py           ★ AI 聊天蓝图：发送消息/历史记录，含 ARK API 调用 + 规则引擎
│   ├── user.py           ★ 用户蓝图：资料查询/修改/头像上传，全部需认证
│   └── resources.py      ★ 资源蓝图：急救/药品/医疗机构查询，公开访问
│
├── uploads/              用户上传的头像文件（Git 忽略）
├── venv/                 Python 虚拟环境（Git 忽略）
└── .gstack/              gstack 浏览器工具配置（与业务无关）
```

> ★ = 关键文件，接手项目优先阅读

## 阅读顺序建议

### 后端（从零理解）
1. `config.py` → 了解有哪些配置项
2. `utils.py` → 了解数据库访问和响应格式
3. `app.py` → 了解蓝图注册和应用结构
4. `auth.py` → 了解认证机制
5. `api/user.py` → 了解 CRUD 模式
6. 其余 `api/*.py` 按需阅读

### 前端（从零理解）
1. `vite.config.js` → 了解代理和别名
2. `src/router/index.js` → 了解路由结构
3. `src/utils/request.js` → 了解 HTTP 层
4. `src/stores/user.js` → 了解认证状态管理
5. `src/api/*.js` → 了解后端对接
6. `src/views/*.vue` 按需阅读
