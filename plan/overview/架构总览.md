# 架构总览

## 整体架构

```
浏览器 (localhost:3000)
  │
  ├── Vue 3 SPA (Vite dev server)
  │     ├── Vue Router (10 routes)
  │     ├── Pinia Stores (user / app / health)
  │     ├── Axios (base: /api/v1)
  │     └── 高德地图 JS API (AMap)
  │
  │  Vite proxy: /api → 127.0.0.1:5000
  │              /uploads → 127.0.0.1:5000
  ▼
Flask (localhost:5000)
  │
  ├── 6 Blueprints (/api/v1/*)
  ├── JWT Auth (flask-jwt-extended)
  ├── Raw SQL via sqlite3
  └── AI: 火山引擎 ARK API (Doubao)
```

## 认证流程

```
1. 注册/登录
   POST /auth/register 或 /auth/login
   → 后端验证密码 (werkzeug check_password_hash)
   → 返回 { user, token }

2. 前端存储
   userStore.saveAuth()
   → sessionStorage: isLoggedIn=true
   → localStorage: unihealth_token, unihealth_user

3. 后续请求
   Axios 请求拦截器自动附加
   → Authorization: Bearer <token>

4. Token 过期 / 401
   Axios 响应拦截器
   → 清除所有 storage 键
   → 用户回到未登录状态
```

## 数据库 ER 图（当前活跃 6 表）

```
users
  id (PK), student_id (UNIQUE), username (UNIQUE), password, email, phone,
  avatar_url, gender, height, weight, created_at, updated_at
  │
  ├── 1:N ── posts
  │           id (PK), user_id (FK→users.id), content, likes, comments,
  │           status, created_at
  │             │
  │             └── 1:N ── post_likes
  │                         post_id (FK→posts.id), user_id (FK→users.id)
  │                         PK(post_id, user_id)
  │
  └── 1:N ── chat_sessions
              id (PK), user_id (FK→users.id), created_at
                │
                └── 1:N ── chat_messages
                            id (PK), session_id (FK→chat_sessions.id),
                            role (CHECK 'user'/'ai'), content, created_at

articles (独立表，无外键)
  id (PK), title, excerpt, content, tag, tag_class, img_url,
  img_alt, views, status, created_at, updated_at

posts
  id (PK), user_id (FK→users.id), author, content, likes, comments,
  status, created_at, updated_at
```

## AI 聊天双层回复机制

```
用户发送消息
  │
  ▼
call_ai_api() — 调用火山引擎 ARK (Doubao)
  ├── 成功 → 返回 AI 回复 (is_rule=false)
  └── 失败/超时/缺配置
        │
        ▼
      match_reply() — 规则引擎关键词匹配
        ├── 危机 → 提供热线 400-161-9995
        ├── 压力 → 共情+深呼吸引导
        ├── 睡眠 → 放下手机+白噪音建议
        ├── 悲伤 → 陪伴共情
        ├── 孤独 → 正常化+邀请对话
        ├── 疲惫 → 区分身累/心累
        └── 无匹配 → 随机通用回复 (is_rule=true)
```

## 前端 Store 使用情况

| Store | 文件 | 实际使用者 |
|-------|------|-----------|
| user.js | 认证状态、token、用户信息 | Login, Register, Profile, Axios 拦截器（Navbar 和 Mental 直接读 localStorage，不走 Store） |
| health.js | 饮水、清单、BMI | **未被任何组件导入**（冗余，Health.vue 自行实现了所有逻辑） |
| app.js | 导航项、功能卡片 | **未被任何组件导入**（冗余） |

## 响应格式约定

所有 API 返回统一格式：
```json
{
  "code": 200,
  "message": "ok",
  "data": { ... }
}
```

列表接口额外包含分页：
```json
{
  "code": 200,
  "message": "ok",
  "data": { "items": [...], "total": 50, "page": 1, "limit": 10 }
}
```
